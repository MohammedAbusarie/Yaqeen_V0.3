What is YaqeenScan?

YaqeenScan is a portable, offline-capable QR-based attendance system for the Egyptian Chinese University (ECU). It runs on a professor’s Windows laptop: students scan a QR code, open the link on their phones, enter ID and name, and submit. The app records attendance locally, blocks duplicate submissions per device and per student, and can export to CSV/XLSX. Designed for LAN use (no internet required) and security (tokens, validation, optional host hardening). Engineered by Mohammed Abusarie; ECU red accent, dark theme.

Session & course management

  • Start session: Professor enters a course name; one active session at a time.
  • End session: Closes the session and triggers automatic export.
  • Session timer: Elapsed time shown in the professor GUI.
  • Stale session recovery: On startup, any leftover active session from a previous run is auto-closed.

QR code & student access

  • Rotating QR code: Time-limited attendance URL; token rotates on a configurable interval (e.g. 100 s) with grace period for in-flight submissions.
  • Click-to-refresh: Professor can click the QR (or URL) to rotate the token and show a new code immediately.
  • Auto-refresh: QR panel and URL update when session/network status changes.
  • Network mode badge: Shows “YaqeenScan Hotspot” or “LAN Fallback” so the professor knows how students should connect.

Student submission flow

  • Token-gated URL: Only valid /attend?token=... links open the form; root and invalid tokens get a bland/404-style response.
  • Attendance form: Student ID (alphanumeric, >6 chars), full name (at least two words), submit button.
  • Success / error pages: Dedicated templates with clear messages. Mobile-friendly, responsive student UI (dark theme).

Anti-fraud & validation

  • Per-student one submission: Same student ID cannot submit twice in the same session.
  • Per-device blocking: After a successful submit, the device (MAC + fingerprint) is blocked from submitting again in that session.
  • Browser fingerprinting and MAC resolution (via ARP) for device identity.
  • Signed “already submitted” cookie (HttpOnly, SameSite=Strict) to prevent reuse of the same browser.
  • Rate limiting, CSRF protection, and input validation with generic user-facing error messages.

Network & connectivity

  • Windows WiFi hotspot: Optional built-in hotspot (SSID/password configurable, e.g. “YaqeenScan”) so the laptop can be the access point; session continues if hotspot fails (LAN fallback).
  • LAN fallback: If the hotspot is unavailable, professor and students can use the same existing LAN; server binds to a random high port.

Professor desktop GUI

Single main window with dark theme (ECU palette), CustomTkinter. Left panel: QR code + URL + network badge. Right panel: live attendance list (row number, student ID, name, submission time) with scroll. Session controls: Start Session (course name dialog), End Session, Export, About. Status bar with hotspot/network status and session timer. About dialog with product description and feature summary.

Export

  • Auto-export on session end: Attendance exported to CSV and XLSX in the configured export folder.
  • Manual export: “Export” button exports the current session’s attendance (same filenames, overwrite).
  • Formats: CSV (UTF-8), XLSX (attendance sheet, header styling, auto column width). Filename from course name (sanitized) and session date (e.g. Course_Name_YYYY-MM-DD.csv / .xlsx).

Platform & deployment

Windows-only. Portable single EXE (e.g. PyInstaller build); templates/static bundled. SQLite DB (e.g. data/yaqeen.db) for sessions, attendance, and blocked devices. Admin (UAC) required for hotspot, firewall, ARP. Optional security hardening: firewall lockdown, NetBIOS/LLMNR disabled, ICS stopped during session; reverted on session end.

About Yaqeen

Yaqeen is a set of tools for educators: automated attendance & grades input, schedule maker, and a downloadable QR deployable app. Everything runs in your browser or as a local tool—no server, no data upload. Built by Mohammed Abusarie, Teaching Assistant at the Egyptian Chinese University.

The name Yaqeen (يَقِين) means certainty—reliable automation you can trust so you can focus on teaching and students.
